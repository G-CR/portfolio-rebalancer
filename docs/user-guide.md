# 用户指南

## 建立组合

先在“资产配置”确认五类资产的目标比例合计为 100%，再到“持仓与成本”添加标的。国内标的使用 CNY，SPY、QQQ 使用 USD；可以为标的选择首选行情来源，也可以跟随全局优先级。

初始成本价是当前持仓的平均交易币种成本价。成本汇率表示形成当前人民币成本时的平均换汇成本，不是当前汇率。基准汇率用于剔除汇率影响，普通追加买入不会自动修改它。

## 维护成本

“追加买入”先生成成本预览，再更新份额、平均成本价和成本汇率。费用可以按默认规则估算，也可以录入券商实际费用；勾选保存默认值后，下次打开同一标的会自动带出。

部分卖出只减少份额，平均成本价和成本汇率保持不变。本系统不计算已实现盈亏。人工修正必须填写备注，历史调整可以作为一笔新的修正恢复，不会删除原审计记录。

## 行情与覆盖

数据源页显示有效值、来源、市场时间、抓取时间和状态。自动获取失败时系统继续使用最近有效值并标记“数据已过期”。手动覆盖必须填写备注，可设置失效时间；覆盖有效期间优先于自动值。

新增持仓后，或进入持仓页时发现当前价、当前汇率缺失，系统会立即自动刷新一次，不需要等待定时任务。刷新期间持仓和成本字段仍然可见；如果自动获取失败，页面会显示安全的来源诊断，并提供“立即重试”和“手动录入”。手动录入会直接打开对应行情或汇率的覆盖表单，且不会修改成本价、成本汇率或份额。

Tushare 和 Alpha Vantage 密钥保存后只显示掩码。供应商测试只记录成功或失败，不回传密钥。

## 总览与盈亏

“实际占比”使用当前汇率折算人民币市值。“剔汇率占比”使用基准汇率，用来判断比例变化中有多少来自汇率。浮动盈亏只针对当前持仓，并满足：浮动盈亏 = 价格影响 + 汇率影响。

历史快照表示各时点的持仓状态，不是严格的时间加权收益率。

## 再平衡

输入本次可投入的人民币和美元，设置允许偏离、最小交易金额、是否允许卖出和换汇。默认使用实际人民币占比，也可以切换剔汇率口径比较建议差异。

“保存方案”只保存数学建议。“开始本次再平衡”创建调整前快照，不会向券商下单。完成实际交易后，先更新系统持仓，再点击“完成再平衡并建立新基准”；系统会创建调整后快照并把当前汇率设为新的基准，但不会改变成本价或成本汇率。

## 备份恢复

运行 `make backup` 创建数据库备份。运行 `make restore FILE=backups/file.dump` 恢复；恢复前系统会停止写入服务并生成安全备份。数据库备份不包含独立的凭据密钥卷，两者都应保留。
